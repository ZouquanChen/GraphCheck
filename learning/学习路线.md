# GraphCheck 项目学习路线

> GraphCheck 是一个 ACL 2025 论文项目，核心思想：用知识图谱（KG）增强 LLM 的事实核查能力。  
> 从文档和声明中提取 KG，用 GNN 编码图结构，再将图表示注入 LLM 进行 support/unsupport 二分类。

提出 GraphCheck，一个单次调用、图增强的事实核查框架，能够：
1. 捕捉长文本中的复杂实体关系；
2. 提高核查精度与效率；
3. 增强可解释性。

### 基础知识了解

#### 1. 知识图谱基础 — 三元组与 KG

核心概念：
- 三元组 (head, relation, tail)：如 (Dr. Erica Pan, is, California state epidemiologist)。
- 知识图谱 (KG)：由大量三元组构成的有向图，节点是实体，边是关系。

在项目中的体现：
- 代码通过 LLM 从文本中抽取三元组（提示词见论文 Appendix H）。
- 构建两个图：G_D（文档图）和 G_C（待核查文本图）。
- 图结构后续输入 GNN 进行编码。

#### 2. 图神经网络 — GAT 与消息传递

核心概念：
1. 消息传递：每个节点聚合邻居信息来更新自身表示。
2. GAT（Graph Attention Network）：引入注意力机制，给不同邻居赋予不同权重。

在项目中的体现：
1. 项目使用了 GATConv 和 TransformerConv（来自 PyTorch Geometric）。
2. GAT 更适合局部关系建模（如 XSum 数据集提升明显）；
3. TransformerConv 则能捕捉全局依赖（在 Summeval、CNN 上更优）。
4. 图编码过程：
   - `v_i^{l+1} = UPDATE(v_i^l, sum_{j∈N_i} MESSAGE(v_j^l, e_ji))`
   - 最终通过 READOUT（如 sum pooling）得到图嵌入 h_g。

#### 3. 大语言模型微调 — Frozen LLM + 可训练投影层

核心概念：
- Frozen LLM：大模型参数固定，只训练少量附加模块，保持原有能力并降低显存消耗。
- 投影层（Projector）：将图嵌入映射到 LLM 的文本嵌入空间，实现多模态融合。

在项目中的体现：
- LLM（如 Llama3.3 70B、Qwen2.5 72B）参数冻结。
- 训练对象仅为 GNN + Projector。
- 最终输入 LLM 的是：`[text_embedding, projected_graph_embedding_D, projected_graph_embedding_C]`
- 单次前向即可得到"支持/不支持"的判断。

#### 4. Sentence Transformers — 节点/边文本编码

核心概念：
- 将文本转换为固定维度的向量（句子嵌入），便于图神经网络处理。
- 项目使用 all-roberta-large-v1，输出维度 1024。

在项目中的体现：
- 每个节点（实体）的文本描述 → 嵌入为 v_i。
- 每条边（关系）的文本描述 → 嵌入为 e_ij。
- 这些嵌入作为 GNN 的初始节点/边特征。

### 第 1 步：数据集结构 — `dataset/*.pkl`

每条数据包含：
- `doc_text`: 原始文档文本
- `claim_text`: 声明文本
- `doc_kg`: 文档的知识图谱三元组列表，如 `[("entity1", "relation", "entity2"), ...]`
- `claim_kg`: 声明的知识图谱三元组列表
- `label`: 标签 (1=support, 0=unsupport)

### 第 2 步：图构建流程 — `graph_build.py`

这是数据预处理的核心，将原始 KG 三元组转为 PyTorch Geometric 的图数据：

- `textualize_graph()`: 将 KG 三元组转为节点表（node_id, node_attr）和边表（src, edge_attr, dst）
- `step_one()`: 遍历所有样本，为每个 claim 和 doc 生成 nodes/edges CSV 文件
- `step_two()`: 用 Sentence Transformer (`all-roberta-large-v1`) 将节点/边文本编码为 1024 维向量，构建 PyG `Data` 对象保存为 `.pt` 文件
- `generate_split()`: 60/20/20 划分 train/val/test 索引

### 第 3 步：数据加载 — `dataset/utils/`

- `modeling.py`: 加载 Sentence Transformer 模型，提供 `load_text2embedding()` 函数
- `dataset.py` (`KGDataset`): 加载 `.pkl` 元数据和 `.pt` 图文件，构造 prompt 文本：
  ```
  Question: Does the Document support the Claim? Please Answer in one word in the form of 'support' or 'unsupport'.
  Claim: {claim}
  Document: {doc}
  ```
- `collate.py`: 用 `Batch.from_data_list()` 将多个图合并为 mini-batch

### 第 4 步：GNN 模型 — `model/gnn.py`

两种 GNN 架构可选：

- **GAT**: 多层 `GATConv`，带 BatchNorm + ReLU + Dropout，返回注意力权重（默认使用）
- **GraphTransformer**: 多层 `TransformerConv`，支持边特征

两者输入输出一致：`(x, edge_index, edge_attr) → (node_embeds, edge_attr)`

### 第 5 步：核心模型 — `model/graphcheck.py`（最重要）

`GraphCheck` 类的结构：

```
GraphCheck
├── model (LLM)          — 冻结的 Llama/Qwen，不参与训练
├── graph_encoder (GNN)  — GAT 或 GraphTransformer，可训练
├── projector (MLP)      — Linear(1024→2048) + Sigmoid + Linear(2048→LLM_dim)，可训练
└── word_embedding       — LLM 的词嵌入层（冻结）
```

关键方法：

- `encode_graphs(data)`: 用 GNN 编码 claim_kg 和 doc_kg，scatter mean 池化得到图级表示
- `forward(data)`: 训练时的前向传播
  - 拼接顺序：`[BOS] + claim_graph_embed + doc_graph_embed + text_embed + label + [EOS]`
  - 计算 language modeling loss（只在 label 部分计算）
- `inference(data)`: 推理时
  - 拼接：`[BOS] + claim_graph_embed + doc_graph_embed + text_embed + [/INST]`
  - 用 `model.generate()` 生成 "support" 或 "unsupport"

### 第 6 步：训练与评估 — `train.py`, `inference.py`, `src/`

- `train.py`: AdamW 优化器 + cosine 学习率调度 + warmup + early stopping + 梯度裁剪
- `inference.py`: 加载 best checkpoint，逐 batch 生成预测并保存为 CSV
- `src/utils.py`: 用 balanced accuracy 评估，正则匹配 "support"/"unsupport"

## 阶段四：深入理解的关键问题

带着这些问题去读代码，理解会更深：

1. **GNN 编码后的图表示是如何注入 LLM 的？**
   → 看 `graphcheck.py` 的 `forward()` 方法，图嵌入被拼接到文本嵌入序列的前面

2. **为什么 LLM 是冻结的？只训练了哪些参数？**
   → 只训练 GNN (`graph_encoder`) 和投影层 (`projector`)，LLM 参数全部冻结

3. **scatter mean 池化做了什么？为什么需要它？**
   → 将同一个图中所有节点的表示取平均，得到一个固定维度的图级表示

4. **模型输出 "support"/"unsupport" 是怎么实现的？**
   → LLM 生成式输出文本，再用正则表达式匹配关键词

5. **Projector 的作用是什么？**
   → 将 GNN 输出维度 (1024) 映射到 LLM 的 embedding 维度，使图表示能与文本 embedding 拼接
